## Scope

- Includes runtime image and audio source files under `assets/`, `music/`, and `sounds/` with common Godot runtime extensions.
- Includes `img/` marketing image/audio files only when a Godot `.import` sidecar exists, then classifies them as `archive` unless later evidence promotes them.

## Classification rules

Priority order:

1. `archive` - `assets/bestiary/` because its README is marked legacy-status `archive`, plus imported `img/` marketing files that are not active runtime candidates.
2. `unknown rights` - all audio, generic `download*` art, and wall/reference renders without source/license evidence.
3. `inconsistent` - legacy HUD/system UI, eight-direction player frame sets, and old tile/isometric materials that conflict with the pending P0-040 art direction.
4. `prototype` - remaining runtime art candidates that may be useful but are not approved and still need P0-028 provenance plus P0-040 style review.
5. `approved` - reserved for future assets with documented source, rights, edits, and human approval. Count is currently zero.
