# Asset Inventory

Generated by `python3 tools/generate_asset_inventory.py` for TODO P0-027, updated after the P0-029 quarantine, and active-path pruning for P0-030.

## Scope

- Includes active runtime image and audio source files under `assets/`, `music/`, and `sounds/` with common Godot runtime extensions.
- Includes `img/` marketing image/audio files only when Godot can import them from the active tree; P0-029 quarantined marketing imports are excluded.
- Excludes `quarantine/` because its `.gdignore` prevents Godot from importing files there; provenance remains in `assets/SOURCES.csv` using mirrored original paths.
- Excludes Godot `*.import` sidecars from the per-file rows because they are metadata, not source image/audio files.
- Uses conservative classifications. Nothing is marked `approved` until provenance, commercial rights, and style approval are documented.

## Classification rules

Priority order:

1. `archive` - `assets/bestiary/` because its README is marked legacy-status `archive`, plus imported `img/` marketing files that are not active runtime candidates.
2. `unknown rights` - unapproved audio, generic `download*` art, and wall/reference renders without source/license evidence.
3. `inconsistent` - legacy HUD/system UI, eight-direction player frame sets, and old tile/isometric materials that conflict with the pending P0-040 art direction.
4. `prototype` - remaining runtime art candidates that may be useful but are not approved and still need P0-028 provenance plus P0-040 style review.
5. `approved` - assets whose `assets/SOURCES.csv` approval begins with `approved` after source, rights, and edits are documented.
